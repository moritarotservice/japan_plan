# 日本計畫：網站驗證總規劃

更新日期：2026-05-14

## 目標

先不做完整 SaaS。第一階段只做可以投廣告、收名單、測需求的網站。

網站的目的不是介紹公司，而是驗證：

- 哪個痛點最能吸引目標客戶
- 哪個 offer 會讓使用者願意輸入真實資料
- 哪個方向有人願意留下 Email / LINE / 預約 demo
- 哪個方向有機會收 setup fee 或早期月費

## 驗證原則

- 不做泛用 landing page，要做具體小工具頁
- 不問「你覺得如何」，直接讓使用者產生一份結果
- 先收真實資料，再判斷需求強度
- 每個工具都要有明確 CTA
- 每個工具都要能追蹤轉換事件

## 第一輪優先順序

1. Salon Guest Check：日本美容 / 沙龍外國客預約前確認工具
2. 外國旅客 FAQ 產生器
3. 多語入住前訊息產生器
4. 多語菜單 QR 產生器
5. 外國客評論回覆產生器
6. 日本對海外採購 / 詢價工具
7. 日本商家外國客 CRM
8. 日本市場商品頁優化器

## 共同網站結構

每個小工具網站都先用同一套結構：

1. Hero：一句話說明痛點與結果
2. Demo form：讓使用者輸入資料
3. Result preview：顯示部分結果
4. Email gate：留下 Email / LINE 才能取得完整結果
5. Next CTA：預約 demo、建立看板、取得導入方案
6. Trust section：說明資料不會公開、結果可人工確認

## 第一輪廣告預算

建議 14 天測試：

- 每個主要方向：NT$10,000-20,000
- 第一輪總預算：NT$20,000-40,000
- 先測日本端商家，再依數據決定是否測台灣端跨境商務

## 核心成功指標

比按讚更重要的行為：

- 使用者完成表單
- 使用者輸入真實業務資料
- 使用者留下 Email / LINE
- 使用者下載結果
- 使用者點擊預約 demo
- 使用者詢問價格
- 使用者願意付 setup fee

## 建議技術做法

第一版不需要完整後台，可以使用：

- 單頁網站
- 表單
- 結果頁
- Email capture
- Google Analytics / Meta Pixel
- Airtable / Google Sheet 收資料
- 人工或半自動產生結果

之後如果某個方向數據好，再開發真正的 SaaS。
